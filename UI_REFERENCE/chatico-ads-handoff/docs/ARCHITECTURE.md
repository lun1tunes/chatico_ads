# ARCHITECTURE.md — Chatico Ads Dashboard (Frontend)

## 1. Назначение документа

Описывает архитектуру фронтенд-части дашборда Chatico Ads — интерфейса, через который владелец малого бизнеса видит результаты своей рекламы (сейчас Meta/Facebook, далее Google Ads и TikTok Ads) и может задать вопрос AI-консультанту.

Бэкенд, база данных, авторизация через площадки (Meta/Google/TikTok), верификация — вне зоны этого документа, делается отдельной командой. Здесь — только интерфейс и его логика.

## 4. Иерархия экранов

```
DashboardPage (Overview, по выбранному аккаунту)
├── PeriodSwitcher (7 / 14 / 30 / 60 / 90 дней)
├── VerdictCard          — AI-вердикт по всему аккаунту, простым языком, 3 предложения макс.
├── KeyMetricsRow        — 4 метрики: потрачено / отклики / цена за отклик / показы
├── TrendChart
├── CampaignList         — список кампаний человеческим языком
└── AIPanel
    ├── ChatWindow
    └── SuggestedChips (1–2 примера вопроса, исчезают при вводе/выборе)

CampaignDetailPage (/campaigns/:id)
├── CampaignSummary      — те же 4 метрики, но по конкретной кампании
└── AdSetList
    └── AdSetRow
        ├── TargetingLine (👤 пол · возраст · город — мелко, для контекста)
        └── условный рендер объявлений:
            ├── 1 объявление    → AdCard сразу, без вложенности
            ├── 2–3 объявления  → AdList (все)
            └── 4+ объявлений   → топ-3 (по цене за отклик) + ShowMoreButton → остальные
```

## 5. Раздел метрик (Meta/Facebook)

### Общая сводка по аккаунту (Overview)

| Метрика (для владельца) | Техническое название | Комментарий |
|---|---|---|
| Сколько потратили | Расход | сумма за период |
| Сколько людей откликнулось | Сообщения / Лиды / Переходы (по цели кампании) | главная метрика результата |
| Цена за один отклик | CPL / Цена результата | ключевая метрика для решения "продолжать/стоп" |
| Сколько раз показали | Показы | вторично, для контекста |

Вторичные метрики (CPM, CPC, охват, частота) — не на главном экране, доступны в деталях кампании.

### Правило разделения целей кампаний

Кампании в Meta имеют одну из 6 целей (Awareness, Traffic, Engagement, Leads, Sales, App Promotion). Сводка **не складывает** разные типы результата в одну цифру:
- если все активные кампании на одну цель (сейчас — преимущественно "Сообщения") — сводка простая, одна цифра;
- если цели разные — показывать раздельно ("Сообщения: X" / "Переходы: Y"), не объединяя в общую сумму.

### Детализация: Кампания → Группа объявлений → Объявление

- Сортировка объявлений внутри группы — по цене за отклик (дешевле выше), лидер помечается ("Лучший результат").
- Таргетинг показывается одной короткой строкой под названием группы (пол · возраст · город) — без раскрытия полных настроек Meta.

## 6. AI-слой (вердикт + чат)

- **VerdictCard** — короткий текст (до 3 предложений): оценка → причина → рекомендация. Пересчитывается при смене периода, обновляется не чаще раза в день.
- **ChatWindow** — свободные вопросы по данным аккаунта.
- **SuggestedChips** — 1–2 примера вопроса. Исчезают навсегда при первом вводе текста или клике на чипс (флаг `hasInteracted`, без возврата при очистке поля).

**Жёсткое правило языка** (для бэкенд-промпта AI, см. `AI_LANGUAGE_RULES.md`): никаких терминов (CTR, CPM, CPC, CPL, CPA, ROAS, "охват", "частота") в текстах вердикта и чата — только смысл и деньги простыми словами.

## 7. Платформы (масштабирование)

```
Sidebar:
├── Facebook/Instagram  — активная реализация
├── Google Ads          — ComingSoonPlaceholder
└── TikTok Ads          — ComingSoonPlaceholder
```

Каждая платформа — отдельный раздел с одинаковой логикой экранов (Overview → Campaign Detail → AI-панель), но со своим набором метрик и целей внутри (продумывается отдельно при подключении).
