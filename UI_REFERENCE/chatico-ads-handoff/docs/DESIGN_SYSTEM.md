## 1. Chatico Brand Overview & Philosophy

**Что такое Chatico.** AI-агент для бизнеса, который превращает WhatsApp, Instagram и Telegram в управляемый канал продаж: обрабатывает диалоги и комментарии, отсеивает спам/хейт, доводит клиента до менеджера и показывает аналитику там, где раньше был "чёрный ящик".

**Миссия.** Сделать AI-сотрудника, который снимает с бизнеса рутину первого контакта с клиентом и возвращает в переписки то, что раньше терялось — лиды, инсайты, время менеджеров.

**Ключевые ценности:**
- **Прозрачность вместо "чёрного ящика"** — "WhatsApp больше не чёрный ящик, это управляемая аналитика продаж"
- **Официальность и безопасность** — верифицированный продукт Meta, работа через официальный API
- **Забота о рутине клиента** — AI берёт на себя первичную обработку, экономит часы живой работы
- **Результат, а не теория** — конкретные цифры и кейсы вместо абстрактных обещаний

**Tone of Voice.** Дружелюбный и прямой, без канцелярита. Короткие фразы-факты, разговорные вопросы-боли, уверенный продающий тон без агрессии. Близко к стилю Google-продуктов: просто, по-человечески, фокус на пользе.

## 2. Design Tokens

### 2.1 Logo System (официально, из файлов)

**Wordmark.** Строчными буквами "chatico" — геометричный гротеск с мягко скруглёнными терминалами. Буква "o" в конце стилизована как речевое облако (значок чата с "хвостиком" внизу слева) — это и есть переход к отдельному icon mark.

**Icon mark / favicon.** Тот же значок речевого облака (indigo) на круглой подложке lime — используется как favicon и app-иконка.

**Система саб-брендов (важно — это уже установленный паттерн).**
Подтверждено файлом `chatico ads`: при создании дополнительного продукта/сервиса под зонтичным брендом —
- слово **"chatico"** остаётся **Indigo** `#5E44EB`
- слово-суффикс продукта (например **"ads"**) пишется тем же шрифтом, но в **Lime** `#DCF252`

Это даёт готовый шаблон для будущих суб-продуктов (например `chatico crm`, `chatico bot` и т.д.) — название всегда из двух слов, разный цвет на каждое.

**Инверсия цветов (правило, не отдельный готовый файл).**
Допустима зеркальная раскладка — "chatico" в Lime, суффикс в Indigo — как альтернативный вариант той же пары цветов:

| Вариант | "chatico" | Суффикс | Когда использовать |
|---|---|---|---|
| Основной | Indigo `#5E44EB` | Lime `#DCF252` | По умолчанию, на светлом/белом фоне — соответствует существующему файлу `chatico ads` |
| Инвертированный | Lime `#DCF252` | Indigo `#5E44EB` | На тёмном/насыщенном Indigo-фоне (где Indigo-текст потеряется), либо как визуальная вариация для разграничения линеек продуктов |

Правила инверсии:
- Внутри одного лого используются **только** эти два цвета — никогда не вводить третий цвет для разделения слов
- Один из двух цветов в паре всегда должен быть тем, что даёт контраст ≥ 4.5:1 к фону (для Indigo на белом — ок; для Lime на белом — текст нечитаем, поэтому Lime-слово в инверсии лучше работает на тёмной подложке, не на белом фоне)
- Не смешивать схему — то есть нельзя сделать "chatico" наполовину Indigo, наполовину Lime в одном слове
- Основной (не инвертированный) вариант остаётся вариантом по умолчанию для всех материалов на белом/светлом фоне; инверсия — осознанный выбор для тёмных тем, сторис/соцсетей с цветным фоном или для визуального различения линейки продуктов друг от друга

> Если нужна именно готовая SVG/PNG-инверсия как файл — это нужно отрисовать отдельно (зеркально перекрасить существующие SVG); я могу сделать SVG-вариант с инверсией fill-значений, если пришлёшь подтверждение, что это нужно как реальный ассет, а не просто правило в документации.

### 2.4 Typography

**Montserrat** — уже используется в реальных презентациях Chatico, поэтому это официальный шрифт бренда, а не предложение с нуля. Geometric sans, строгий и читаемый — даёт ту самую "чистоту Google", которая нужна по ТЗ. Округлость и дружелюбие бренд уже получает через лого (круглая "o"-чат-иконка) — шрифту не нужно её дублировать, контраст "строгий текст + округлая иконка" работает лучше, чем два декоративных шрифта подряд.

| Стиль | Размер | Line-height | Letter-spacing | Вес |
|---|---|---|---|---|
| Display | 48–64px | 1.05 | -0.02em | 700 (Bold) |
| Headline | 28–32px | 1.2 | -0.01em | 700 (Bold) |
| Title | 18–20px | 1.3 | 0 | 600 (SemiBold) |
| Body | 14–16px | 1.5 | 0 | 400–500 (Regular/Medium) |
| Label / Caption | 12–13px | 1.4 | 0.01em | 500 (Medium) |

**Подключение (Google Fonts):**
```html
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@400;500;600;700&display=swap" rel="stylesheet">
```

### 2.5 Spacing & Radii

- Базовая сетка: **8px**
- Шаги: 4 / 8 / 16 / 24 / 32 / 48 / 64px
- Радиусы: кнопки/инпуты/бейджи — `12px`; карточки — `16px`; крупные блоки/модалки — `24px`; круглая подложка иконки — `50%` (как в favicon)
- Тени мягкие: `0 2px 8px rgba(28,27,31,0.06)` в покое, `0 4px 16px rgba(28,27,31,0.10)` на hover

## 3. Brand Mascot & Characters

**Кто это.** Низкополигональный (low-poly) 3D-персонаж — деловой человек: зелёные "огранённые" волосы, тёмные прямоугольные очки, светлая кожа, индиго-синий деловой костюм с белой рубашкой и зелёным галстуком.

**Визуальный стиль:** геометричный low-poly, плоские заливки с лёгким градиентным шейдингом, дружелюбно-карикатурный, не реалистичный.

**AI Integration Rules — описание для генерации новых изображений:**
> "Low-poly 3D character, faceted geometric style, green crystalline hair, dark rectangular glasses, light skin tone, wearing a navy-indigo business suit with white shirt and lime-green tie, friendly confident expression, [поза/атрибут], soft gradient lighting, isolated on transparent or solid background"

**Дозированное использование в UI:**
- ДА: онбординг, пустые состояния, уведомления об успехе/ошибке, маркетинговые лендинги
- НЕТ: рабочие экраны с данными (таблицы, дашборды, настройки) — там это лишний визуальный шум
- Маскот никогда не используется как иконка кнопки или элемент навигации

## 4. UI Components & Layout Guidelines

### Buttons
- **Primary:** фон `--color-primary` (Indigo), белый текст, radius 12px
  - Hover: затемнение на 8%; Focus: outline 2px Indigo, offset 2px
- **Secondary/Outline:** прозрачный фон, рамка 1px `--color-outline`
- **Текстовая кнопка:** без фона, цвет Indigo, подчёркивание на hover

### Cards
- Фон белый, radius 16px, тень `0 2px 8px rgba(0,0,0,0.06)`, паддинг 16–24px

### Badges/Status Chips
- Высота 24–28px, radius 12px (pill), фон = `*-container`-токен статуса, текст — насыщенный тон того же цвета

### Inputs
- Рамка 1px Outline, radius 12px; focus — рамка Indigo + тень `0 0 0 3px` с alpha 10%

### Navigation Bar
- Светлый фон, тонкая граница-разделитель 1px снизу
- Активный пункт — текст Indigo + точечная подложка `--color-primary-container`

### Brand mark в интерфейсе
- Favicon/app-иконка — речевое облако Indigo на круглой Lime-подложке (по официальному файлу)
- В хедере/навигации — текстовый wordmark "chatico" (Indigo), без круглой подложки

## 5. Do's and Don'ts

**ДА:**
- Использовать официальные Indigo `#5E44EB` и Lime `#DCF252` как единственную базовую пару бренд-цветов
- Для суб-продуктов — называть по схеме "chatico + [слово]" с разделением цвета (chatico = Indigo, суффикс = Lime)
- Разделять блоки цветом подложки, а не рамками
- Оставлять много пустого пространства
- Использовать маскота только в эмоциональных "пустых" точках интерфейса

**НЕТ:**
- Не использовать Lime как цвет текста на светлом фоне (низкий контраст)
- Не обводить карточки чёрными/тёмно-серыми жирными рамками
- Не размещать маскота на экранах с таблицами и графиками
- Не делать радиус скругления меньше 8px
- Не придумывать новые бренд-цвета помимо Indigo/Lime без согласования — это официальная пара из лого-кита
